In Markdown, a checklist (task list) is created using a hyphen `-`, a space, and square brackets.

### Unchecked Items

```markdown
- [ ] Learn Markdown
- [ ] Build a Portfolio
- [ ] Push Project to GitHub
```

Result:

* [ ] Learn Markdown
* [ ] Build a Portfolio
* [ ] Push Project to GitHub

---

### Checked Items

```markdown
- [x] Learn Markdown
- [x] Create README
- [ ] Deploy Project
```

Result:

* [x] Learn Markdown
* [x] Create README
* [ ] Deploy Project

---

### Mixed Example

```markdown
## Learning Roadmap

- [x] Headings
- [x] Paragraphs
- [x] Lists
- [x] Images
- [ ] Tables
- [ ] GitHub Actions
- [ ] CI/CD
```

Result:

## Learning Roadmap

* [x] Headings
* [x] Paragraphs
* [x] Lists
* [x] Images
* [ ] Tables
* [ ] GitHub Actions
* [ ] CI/CD

### Syntax Breakdown

```markdown
- [ ] Unchecked task
- [x] Checked task
```

This works especially well in GitHub README files, Issues, and Pull Requests, where checkboxes can often be toggled interactively.
